## 1. 背景与目标

### 1.1 背景
用户需要一个工具来管理日常作业/任务，通过积分激励机制提高完成积极性。市面上现有的任务管理工具缺少积分和兑换机制，无法形成完整的激励闭环。

### 1.2 目标
开发一款微信小程序，提供作业管理、积分获取、奖品兑换的完整功能，帮助用户养成良好习惯，提高任务完成率。

### 1.3 范围
**包含：**
- 用户注册/登录
- 作业的增删改查
- 作业完成打卡
- 积分获取与统计
- 奖品管理与兑换
- 积分流水与兑换记录

**不包含：**
- 社交功能
- 第三方登录（一期）
- 实物奖品发货（一期）

## 2. 用户故事

### 故事 1：添加作业
```
作为一个 用户
我想要 添加作业并设置积分
以便于 记录我需要完成的任务并获得激励
```

**验收标准：**
- [ ] 可以输入作业名称（必填，最多20字）
- [ ] 可以输入作业描述（选填，最多100字）
- [ ] 可以选择作业类型（每日/每周/每月/自定义）
- [ ] 可以设置完成获得的积分（必填，正整数）
- [ ] 可以设置未完成惩罚积分（选填）
- [ ] 可以设置积分过期时间（选填）

### 故事 2：完成作业打卡
```
作为一个 用户
我想要 完成作业打卡获得积分
以便于 累积积分兑换奖品
```

**验收标准：**
- [ ] 在作业详情页可以点击"完成打卡"
- [ ] 打卡成功后积分立即增加
- [ ] 当天已打卡的作业显示"已完成"状态
- [ ] 打卡记录保存到历史

### 故事 3：查看积分
```
作为一个 用户
我想要 查看我的积分统计
以便于 了解我的积分获取情况
```

**验收标准：**
- [ ] 显示总积分余额
- [ ] 显示今日获得积分
- [ ] 显示本月获得积分
- [ ] 可以查看积分流水明细

### 故事 4：兑换奖品
```
作为一个 用户
我想要 用积分兑换奖品
以便于 获得完成作业的奖励
```

**验收标准：**
- [ ] 可以查看所有可兑换奖品
- [ ] 奖品显示名称、所需积分、剩余库存
- [ ] 积分不足时无法兑换
- [ ] 兑换成功后积分立即扣减
- [ ] 兑换记录可查看

### 故事 5：管理员管理奖品
```
作为一个 管理员
我想要 添加和管理奖品
以便于 用户可以兑换奖励
```

**验收标准：**
- [ ] 可以添加奖品（名称、所需积分、库存）
- [ ] 可以编辑奖品信息
- [ ] 可以下架奖品

### 故事 6：查看作业日历
```
作为一个 用户
我想要 在日历上查看我的作业安排
以便于 规划我的时间
```

**验收标准：**
- [ ] 日历视图显示每天待完成的作业
- [ ] 已完成/未完成状态清晰展示
- [ ] 可以切换月份查看

### 3.2 功能详细说明

#### 3.2.1 用户注册/登录

**描述：** 使用微信授权快速注册登录

**输入：**
- 微信授权信息：必填

**输出：**
- 用户ID
- 登录Token

**业务规则：**
1. 首次登录自动注册
2. 获取微信昵称和头像作为默认信息
3. 用户可选择成为管理员（需邀请码）

**异常处理：**
- 授权失败：提示重新授权
- 网络异常：提示检查网络

---

#### 3.2.2 添加作业

**描述：** 创建新的作业项

**输入：**
| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 作业名称 | string | Y | 1-20字符 |
| 作业描述 | string | N | 最多100字符 |
| 作业类型 | enum | Y | daily/weekly/monthly/custom |
| 类型配置 | object | Y | 根据类型不同 |
| 完成积分 | number | Y | 正整数，1-1000 |
| 惩罚积分 | number | N | 非负整数，0-500 |
| 积分过期天数 | number | N | 正整数，不填则永不过期 |

**类型配置说明：**
- daily：无额外配置
- weekly：选择周几执行（1-7，可多选）
- monthly：选择每月几号执行（1-28，可多选）
- custom：开始日期、结束日期

**输出：**
- 作业ID
- 创建成功提示

**业务规则：**
1. 同一用户作业名称不可重复
2. 每日作业每天只能打卡一次
3. 每周作业每周每个周期只能打卡一次
4. 自定义作业需在有效期内打卡

---

#### 3.2.3 完成打卡

**描述：** 标记作业完成，获得积分

**输入：**
- 作业ID：必填
- 完成日期：默认今天

**输出：**
- 获得积分数
- 当前总积分

**业务规则：**
1. 每个作业每天最多打卡一次（daily类型）
2. 打卡后积分立即到账
3. 打卡记录不可撤销
4. 补打卡需要管理员权限

**异常处理：**
- 已打卡：提示"今日已完成"
- 作业不存在：提示错误
- 作业已过期：提示无法打卡

---

#### 3.2.4 积分统计

**描述：** 展示用户积分概况

**输入：**
- 用户ID

**输出：**
| 字段 | 说明 |
|------|------|
| 总积分 | 当前可用积分 |
| 今日获得 | 今日打卡获得的积分 |
| 本月获得 | 本月打卡获得的积分 |
| 累计获得 | 历史总共获得的积分 |
| 累计消耗 | 兑换奖品消耗的积分 |
| 即将过期 | 7天内将过期的积分 |

---

#### 3.2.5 积分流水

**描述：** 查看积分变动明细

**输入：**
- 用户ID
- 页码（分页）
- 筛选条件（可选）：类型（获得/消耗）、时间范围

**输出：**
| 字段 | 说明 |
|------|------|
| 流水ID | 唯一标识 |
| 类型 | 获得/消耗/过期/调整 |
| 金额 | 变动积分（正数为获得，负数为消耗） |
| 来源 | 作业打卡/奖品兑换/积分过期/管理员调整 |
| 来源ID | 关联的作业ID或奖品ID |
| 时间 | 创建时间 |
| 备注 | 说明文字 |

---

#### 3.2.6 奖品管理

**描述：** 管理员添加/编辑/删除奖品

**输入（添加）：**
| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 奖品名称 | string | Y | 1-30字符 |
| 奖品描述 | string | N | 最多200字符 |
| 奖品图片 | string | N | 图片URL |
| 所需积分 | number | Y | 正整数 |
| 库存数量 | number | Y | 非负整数 |
| 状态 | enum | Y | 上架/下架 |

**业务规则：**
1. 管理员权限才能操作
2. 下架奖品用户不可见
3. 库存为0时自动标记为已售罄

---

#### 3.2.7 奖品兑换

**描述：** 用户使用积分兑换奖品

**输入：**
- 奖品ID：必填
- 兑换数量：默认1

**输出：**
- 兑换结果
- 剩余积分

**业务规则：**
1. 积分必须充足
2. 奖品必须上架且有库存
3. 兑换成功立即扣减积分和库存
4. 生成兑换记录

**异常处理：**
- 积分不足：提示"积分不足，还需XXX积分"
- 库存不足：提示"奖品已售罄"
- 奖品下架：提示"奖品不可兑换"

---

#### 3.2.8 兑换记录

**描述：** 查看用户的兑换历史

**输入：**
- 用户ID
- 页码（分页）

**输出：**
| 字段 | 说明 |
|------|------|
| 记录ID | 唯一标识 |
| 奖品名称 | 兑换的奖品 |
| 奖品图片 | 奖品图片 |
| 消耗积分 | 兑换消耗的积分 |
| 兑换时间 | 兑换时间 |
| 状态 | 待发放/已发放/已取消 |

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间：< 2秒
- 接口响应时间：< 500ms（P99）
- 支持并发用户：1000+

### 4.2 安全要求
- 用户数据隔离，只能查看自己的数据
- 管理员权限严格校验
- 敏感操作需要二次确认
- 接口防刷限流

### 4.3 兼容性
- 微信小程序基础库版本：2.20.0+
- 支持 iOS 和 Android

### 4.4 数据安全
- 用户数据每日备份
- 积分变动有完整流水记录
- 支持数据回滚

## 6. 交互设计

### 6.1 页面结构
```
├── 首页（作业列表）
│   ├── 今日作业
│   ├── 作业日历
│   └── 添加作业按钮
│
├── 积分页
│   ├── 积分概况
│   ├── 积分流水
│   └── 兑换入口
│
├── 兑换页
│   ├── 奖品列表
│   └── 兑换记录
│
└── 我的页
    ├── 个人信息
    ├── 我的作业
    └── 设置
```

### 6.2 关键交互流程
1. **添加作业流程**：首页 → 点击添加 → 填写信息 → 保存
2. **完成打卡流程**：首页 → 点击作业 → 点击完成 → 积分增加动画
3. **兑换奖品流程**：积分页 → 点击兑换 → 选择奖品 → 确认兑换
